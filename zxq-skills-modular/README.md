# 强哥目标系统模块化 Skill 架构

> 基于邹小强《只管去做》方法论，从愿景到每日行动的完整目标管理系统

## 📦 架构说明

本项目采用**模块化 skill 架构**，将强哥目标系统拆分成：
- **1 个总控 skill** - 负责路由和流程控制
- **1 个人设底座** - 提供强哥的思维框架和表达风格
- **10 个业务 skill** - 各自负责一个具体功能
- **1 个工具 skill** - miniapp-upload（小程序上传）

## 🎯 总控 skill 职责

`zg-orchestrator` 是整个系统的**路由器和流程控制器**，它只做 4 件事：

1. **听懂用户意图** - 判断用户想做什么
2. **路由到对应 skill** - 调用相应的业务 skill
3. **管理状态** - 维护用户当前进度
4. **承接结果** - 在 skill 之间做自然过渡

**总控不做具体业务**：
- ❌ 不写五年后的信
- ❌ 不生成目标九宫格
- ❌ 不做日计划
- ❌ 不做强哥答疑

---

## 🧠 人设底座的作用

`zouxiaoqiang-perspective` 是所有 skill 的**共享人设层**：

- 提供强哥的思维框架（7个核心心智模型）
- 提供强哥的表达风格（语言 DNA）
- 提供案例库（目标九宫格、衣柜整理法、拖延4类型）
- 提供深度方法论（只管去做、小强升职记等）

所有业务 skill 启动时，会自动加载这一层，确保"说话像强哥"。

## 🔄 工作流示例

### 示例 1：用户想做年度目标

```
用户：我想做年度目标

总控：可以，不过我提醒一句：年度目标最好从"五年后的信"里孵化出来。
     你想：
     A. 先写五年后的信（推荐）
     B. 直接做年度目标（快速版）

用户：A

总控：好，那我们先进入第一步：五年后的信。
     [调用 five-year-letter skill]

[five-year-letter 完成]

总控：好，这封五年后的信我们先定下来。
     下一步，我们把它孵化成年度目标九宫格。
     [调用 goal-grid-from-letter skill]
```

### 示例 2：用户问问题

```
用户：我最近很拖延，总是做不动，强哥怎么看？

总控：好，这个问题我们走强哥答疑。
     [调用 qiangge-qa skill]

[qiangge-qa 接管]

答疑：我听到了。
     你现在不是"不够自律"，而是可能目标太多、能量不够，或者系统没给自己留余地。
     我们先不急着给方法，先看看你真正卡在哪里。
     这个目标，是你真想做的，还是觉得应该做的？
```

## 🔧 工具 skill

- 年度目标九宫格生成后 → 上传到小程序

## 📊 架构优势

### ✅ 职责清晰
- 总控只做路由，不做业务
- 每个业务 skill 独立完整
- 人设底座作为共享层

### ✅ 可维护性强
- 改某个功能只需改对应 skill
- 总控轻量（约 500 行），不会变成巨无霸
- 每个 skill 可以独立测试

### ✅ 可扩展性好
- 加新功能只需加新 skill
- 不需要动总控的核心逻辑
- 可以单独使用某个 skill

### ✅ 用户友好
- 用户可以直接跳到任何一步
- 不强制走完整流程
- 总控会根据状态自然引导

## 🚀 使用方式

### 从总控开始（推荐）
```
你好，我想做年度目标
```

总控会判断你的意图，引导你到合适的 skill。

### 直接进入某个 skill
```
我想写五年后的信
我想做月目标
今天帮我整理待办
强哥，我最近很拖延
```

Claude 会根据触发词自动调用对应的 skill。
