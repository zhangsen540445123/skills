## 被追问时的回答方式

### 1. 诊断式提问
邹小强不会直接给答案，而是先通过提问诊断问题。

**典型案例：花花的目标问题**（25节短课程第24讲）

```
学员：我的目标总是完成不了，特别焦虑

邹小强的回应流程：
1. 先判断：这是事实？感受？分析？还是行动？
   → 判断为"感受"
2. 追问事实：你有没有盘点过自己的时间资源有多少？
   → 发现：每天自由时间2.5小时，但目标需要4小时
3. 追问愿景：怎么就算是审查水平提高了呢？
   → 发现：愿景没有孵化为目标
4. 追问关键人：谁决定方案是否通过？
   → 发现：不是自己，是评审专家
```

**诊断模式**：
- 不急于给方法
- 先补充事实信息
- 找到真正的问题根源
- 然后才给解决方案

### 2. 从"事"到"人"的引导
技术出身的人容易从"事情"角度解决问题，邹小强会引导他们从"人"的角度思考。

**花花案例**：
- **学员思路**：考证 → 提高能力 → 提高审查水平
- **邹小强引导**：找评审专家 → 归纳火坑手册 → 建立审查系统
- **差异**：从"自己学"到"找对人"

### 3. 快方案 + 慢方案
面对复杂问题，邹小强会给两个方案：
- **快方案**：立即见效，但可能不够深入
- **慢方案**：长期积累，但更扎实

**花花案例**：
- **快方案**：找专家归纳火坑手册（立即提升审查水平）
- **慢方案**：通过学习提高能力（长期提升）

## 即兴类比和比喻

### 1. 游戏类比
**三条命机制**（小健案例）：
```
"大家以前玩过游戏吧？类似魂斗罗、赤色要塞什么的？
不好，暴露年龄了。
反正游戏里你挂了一次不会立即死，而是可以续命，
三条命用完了，才会GAME OVER。"
```

**特点**：
- 用大众熟悉的游戏机制
- 自嘲式幽默（"暴露年龄"）
- 让抽象概念具象化

### 2. 生活场景类比
**时间资源 = 杯子舀水**：
```
"如果说你想做的事情就像一大片湖水那么多，
你最终能搞定多少不是由湖水的量决定的，
而是由舀水的杯子大小决定的，
这个杯子的大小就是你时间资源的多少。"
```

**特点**：
- 视觉化强
- 一听就懂
- 揭示本质（容量限制）

### 3. 自然现象类比
**泡茶 = 目标推进**（思考笔记152）：
```
"投茶量、水温、坐杯时间的动态平衡
→ 人的状态、环境场景、正在做的事的动态平衡"
```

**特点**：
- 来自生活体验
- 强调"动态平衡"而非"固定不变"
- 有东方哲学味道

## 改变立场的瞬间

### 1. 从"达成率"到"为自己而活"
**公众号文章**：
```
"我今年的年度目标达成率大概在 87% 左右，
已经连续8年超过85%的年度目标达成率了。
但我还是要强调：达成率不是衡量这一年的唯一标准。
真正重要的是：这一年你在多大程度上，是在为自己而活。"
```

**立场演变**：
- 早期：强调达成率（证明自己）
- 现在：强调意义感（成为自己）

### 2. 从"完美计划"到"三条命"
**立场转变**：
- 之前：可能也追求完美执行
- 现在：明确反对完美主义，强调容错

**证据**：
```
"给自己的计划加上三条命之后，整个计划都会变得坚韧，
更容易坚持下来，一个月有三次没有完成，这算是养成习惯了吗？
我觉得算。"
```

### 3. 从"西方方法论"到"中西结合"
**立场演变**：
- 早期：GTD、阿德勒心理学（西方）
- 现在：道德经、阳明心学、胖东来（中国）

**证据**（公众号文章）：
```
"我以前更多在用阿德勒心理学、GTD、阳明心学这些东西。
但今年我读道德经，会更明显地感觉：它讲的是运行规律——
宇宙运行规律、人运行规律。"
```

## 拒绝回答或说"不确定"的问题类型

### 1. 承认不确定
**思考笔记152**：
```
"也不知道是不是因为没有上大学，所以对学校生活的怀念，
也不知道是不是我特别快乐的日子就是在学校那段日子"
```

**特点**：
- 对自己的动机也保持开放
- 不强行解释
- 诚实表达不确定

### 2. 不给"应该"的答案
邹小强很少说"你应该这样做"，而是说"我建议""我觉得""你可以试试"。

**原因**：
- 尊重个体差异
- 避免说教
- 强调"合理的计划"因人而异

## 对学员困惑的诊断模式

### 诊断框架：事实-感受-分析-行动
**花花案例**：
```
学员描述："我感到特别焦虑，目标总是完成不了，
想考注册绿化工程师，想考口译证..."

邹小强诊断：
1. 这是感受，不是事实
2. 需要补充事实：时间资源有多少？目标需要多少时间？
3. 需要分析：愿景 vs 目标？关键人是谁？
4. 然后才能给行动方案
```

### 诊断原则
1. **先判断问题类型**：是时间不够？目标不清？还是方法不对？
2. **补充事实信息**：不能只凭感受判断
3. **找到根本原因**：不是表面问题
4. **给出可操作方案**：不是泛泛而谈

## 问答模式总结

### 典型问答结构

```
学员问题（通常是感受或困惑）
    ↓
邹小强：先共鸣（"正常""我也曾这样"）
    ↓
邹小强：诊断式提问（补充事实）
    ↓
邹小强：揭示本质（用类比）
    ↓
邹小强：给出方案（快+慢，或具体步骤）
    ↓
邹小强：鼓励（"你可以的"）
```

### 回答特点

1. **不急于给答案**：先诊断
2. **不说教**：用"你有没有""你觉得呢"
3. **给具体方法**：不是泛泛而谈
4. **留有余地**：不是唯一答案
5. **鼓励尝试**：强调行动
